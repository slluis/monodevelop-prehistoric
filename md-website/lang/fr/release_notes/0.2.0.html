MonoDevelop 0.2

MonoDevelop 0.2 est sorti. MonoDevelop est un IDE GNOME destiné à C# dans un premier temps et à d'autres langages .NET dans le futur. Vous pouvez toujours trouver des informations à jour ici. Vous pouvez trouver des didacticiels intéressants ici.

Installer MonoDevelop

Configuration requise

Il vous faut la dernière version de mono avec support ICU.

Vous pouvez télécharger MonoDevelop à cette adresse. Le site de mono peut proposer des rmps pour votre distribution.

Si vous décidez de constuire MonoDevelop à partir d'un tarball, utilisez les commandes suivantes.

	tar zxvf monodevelop-0.2.tar.gz
	cd monodevelop-0.2
	./configure
	make
	make install

Par défaut, MonoDevelop s'installera avec le préfixe /usr/local.

MonoDevelop devrait apparaître dans le menu Applications, de plus, un script shell 'monodevelop' est installé dans le répertoire $(prefix)/bin.

Nouveautés

Intégration Beta du Debugger

Ce release contient le premier aperçu de l'encapsulation du debugger GUI pour MonoDevelop. Il est actuellement en version beta, et il comporte encore certains bugs. L'installation de MonoDebugger 0.6 est nécessaire à son fonctionnement. Actuellement, le Debugger requiert NPTL, c'est donc un paquetage optionnel.

Intégration de Monodoc

Vous pouvez maintenant parcourir la documentation de Monodoc à l'intérieur de MonoDevelop. Son support est actuellement assez limité, mais nous travaillons à une incorporation de plus bas niveau pour permettre une documentation à la demande.

Complétion de code

L'association des touches Ctrl+Espace est maintenant implémentée. Cela signifie que lorsque vous désactivez la complétion de code, l'utilisation de ces touches la réactive à la demande. S'il y a un mot correspondant, le texte sera complété automatiquement.

Les améliorations de l'analyseur syntaxique de SharpDevelop ont été intégrées à ce release. Cela permet une meilleure analyse syntaxique et incorpore des corrections permettant l'utilisation de System. pour lister les types et les espaces de noms.

Explorateur de classes

De nombreux bugs de l'explorateur de classes ont été corrigés, le rendant ainsi nettement plus utile. Un problème avec GTK+ 2.4 a aussi été solutionné, de cette manière, l'explorateur de classes (et MonoDevelop) peuvent fonctionner correctement à la fois avec GTK+ 2.2 et 2.4.

Visionneur HTML intégré

Le visionneur HTML intégré détecte, maintenant, lorsqu'un fichier html est chargé et permet l'affichage simultané du HTML et du résultat en cours de frappe.

Support de projets Java

MonoDevelop supporte maintenant les projets Java, leur compilation et leur exécution. Il n'y a pas encore d'analyseur syntaxique (donc pas de complétion de code), et il n'a pas fait l'objet de beaucoup de tests.

Futures fonctionnalités

Support de Projets avancé

Beaucoup de discussions ont abouti à ce document beta qui décrit un nouvelle manière de construire. Notez qu'il n'est pas question de rompre la compatibilité avec SharpDevelop ou avec les projets SharpDevelop.
