Installer MonoDevelop

Configuration

Vous avez besoin de la dernière version de mono avec le support ICU.

Vous pouvez télécharger MonoDevelop 0.1 ici. Le site de mono peut également disposer des paquetages pour votre distribution.

Si vous décidez de contruire MonoDevelop à partir d'un tarball, vous pouvez le construire en suivant les instructions suivantes. 

	tar zxvf monodevelop-0.1.tar.gz
	cd monodevelop-0.1
	./configure
	make
	make install

Par Défaut, MonoDevelop s'installera avec le préfixe /usr/local

MonoDevelop devrait apparaître dans le menu des Applications et un script shell 'monodevelop' est installé dans le répertoire $(prefix)/bin.

Fonctionnalités

Editeur de texte

MonoDevelop dispose d'une version améliorée de gtksourceview pour un éditeur de texte. Nous l'avons étendu à l'intérieur du code de base de MonoDevelop en utilisant C#. Actuellement, il supporte entre autre la Colorisation Syntaxique, la Complétion de Code, et des gabarits de code.

Support de projets

MonoDevelop possède un support de projets assez complet. Il supporte la création de projets multi-projets, la compilation à l'intérieur de l'éditeur, l'exécution depuis l'éditeur. Le format de projet est actuellement compatible avec celui utilisé par SharpDevelop.

Explorateur de classes

Lorsqu'un projet est ouvert, MonoDevelop identifie automatiquement les classes, méthodes et propriétés disponibles dans ce projet, et génère une arborescence que l'on peut explorer.

Recherche

MonoDevelop propose un outil de recherche/remplacement avancé. Il supporte la recherche dans un fichier, tous les documents ouverts, tous les documents dans un dossier et tous les documents d'un projet. Il supporte également une variété d'options de recherche comme les expressions régulières.

Ancrage des widgets

MonoDevelop utilise une librairie C appelée gdl-dock (disponible actuellement sur le serveur CVS de gnome.org). Cela permet de réaliser un ancrage avancé impressionant, incluant le déplacement des éléments ancrés dans une nouvelle fenêtre, la possibilité de les empiler les uns sur les autres. Gdl-dock permet aussi le masquage et la réduction en icône des éléments.

Visionneur HTML intégré

MonoDevelop dispose d'un visionneur html intégré grâce à gtkmozembed-sharp. Cette fonctionnalité sera finalement utilisée pour permettre le développement ASP.NET.

Futures fonctionnalités

Debugging

Un travail d'intégration du debugger mono à MonoDevelop est actuellement en cours en vue d'offrir un degugging intégré.

Support de projets avancé

Nous travaillons à l'implémentation de la possibilité d'utiliser les composants autotools standards afin de faciliter le processus de construction et de distribution.

Complétion de code avancée

Il est prévu d'améliorer infiniment notre complétion de code. Nous sommes encore en train de spéculer sur la meilleure approche possible pour réaliser ce travail.
