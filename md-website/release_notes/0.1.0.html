MonoDevelop 0.1

Well, it's finally time for the first MonoDevelop release. You can find always current info here. There is also a nice tutorial here.

Installing MonoDevelop

Getting setup

You will need the latest version of mono built with ICU support.

You can download MonoDevelop 0.1 from here. The go-mono site might also have rpms for your distro.

If you decide to build MonoDevelop from a tarball, you can build it with the following command lines.

	tar zxvf monodevelop-0.1.tar.gz
	cd monodevelop-0.1
	./configure
	make
	make install

By default MonoDevelop will install with a /usr/local prefix.

MonoDevelop should now appear in your Applications menu, and a `monodevelop' shell script is installed into your $(prefix)/bin directory.

Features

Text Editor

MonoDevelop has a customized version of gtksourceview for a text editor. We have extended it inside the MonoDevelop codebase using C#. It currently supports Syntax Highlighting, Code Completion, and Code Templates among other features.

Project Support

MonoDevelop also has fairly comprehensive project support. It supports creating multi-project projects, compilation from inside the editor, execution from inside the editor. The project format is currently compatible with the SharpDevelop project format.

Class Browser

When you have a project open, MonoDevelop will automatically identify the project's classes, methods, and properties available in your project, and create a tree for you to browse.

Searching

MonoDevelop has an advanced find/replace setup. It supports finding across one file, all open documents, all documents in a directory, and all documents in a project. It also supports various permutations of finding, including wildcard and regular expressions.

Docking Widgetry

MonoDevelop uses a C library called gdl-dock (current exists in gnome.org's cvs). This allows us to do some fairly advanced docking, including dragging dock items into a new window, dragging them on top of each other. Gdl-dock also supports hiding and iconifying docked items.

Embedded HTML viewer

MonoDevelop currently supports an embedded html viewer thanks to gtkmozembed-sharp. Eventually this functionality will be used to facilitate ASP.NET work.

Coming Soon

Debugging

Work is underway to embed the mono debugger into MonoDevelop and offer integrated debugging.

Enhanced Project Support

We are working towards adding the ability to use standard autotools components to ease the build and distribution process.

Enhanced Code Completion

There is a plan in the works to vastly improve our code completion. We are still doing some speculative work as to what the best approach is.
