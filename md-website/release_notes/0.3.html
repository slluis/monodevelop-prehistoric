MonoDevelop 0.3

MonoDevelop 0.3 has been released. MonoDevelop is a GNOME IDE primarily designed for C# and other .NET languages. You can always find current info here. There is also a nice tutorial here.

Installing MonoDevelop

Getting setup

You can get detailed instructions on how to build MonoDevelop 0.3 from here.

You can download MonoDevelop 0.3 source from here. The go-mono site may contain packages for your distro.

New Features

Gettext support

MonoDevelop now contains the ability to be fully translated like any other GNOME application. Currently we are shipping french and japanese translations, with more to come soon.

Monodoc Integration

Monodoc documentation is now built into your code completion database to provide documentation in tooltips, and in other varied locations.

Code Completion

Numerous enhancements have been put into code completion for this release. Some of them are graphical (XCode style type ahead), however most are functional.

The Control+Space keybinding has been greatly improved in this release. Local variables are now included in the completion list.

Java project support

MonoDevelop's support of Java projects has greatly improved. We now use ikvm and gcj by default to write ECMA compliant Common Language Runtime applications (including GNOME applications via Gtk#) in the Java language, using completely free software and instead of depending on an external compiler and runtime. Ikvm is expected to be included with the 0.32 mono release, otherwise it will require some manual setup. See here for more information.

Nemerle project support

MonoDevelop now includes support for Nemerle projects.

ILAsm project support

Prelimary support for ILAsm projects.

Coming Soon

Enhanced Project Support

There has been some discussion, which has resulted in this beta document to start describing a new build setup. Note, there is no intention of breaking compatibility with SharpDevelop or SharpDevelop projects.
